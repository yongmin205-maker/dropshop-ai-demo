# 스마트 슬롯 추천 — 미용실 AI

> 친구가 1분 안에 읽고 "아 이거구나" 할 수 있게 만든 한 페이지 설명.

---

## 1. 지금 미용실 앱들이 가진 문제

지금 친구 앱 포함 대부분의 예약 앱은 **15분 단위로 가능한 시작 시간을 전부 다 보여줌**.

오전에 이미 잡힌 예약이 이렇다고 가정:

```
10:00  ████████████████████  손님 A · 미디엄 컷 (60분)
11:00  · · · · · · · · · ·   비어있음
12:00  · · · · · · · · · ·   비어있음
12:30  ████████████          손님 B · 숏컷 (45분)
13:15  · · · · · · · · · ·   비어있음
14:00  ██████████████████████  손님 C · 컬러 (90분)
15:30  · · · · · · · · · ·   비어있음
```

여기에 새 손님이 문자로 "오늘 숏컷 45분 예약 가능?"이라고 물어봄.

**현재 앱이 보여주는 옵션:** 11:00 / 11:15 / 11:30 / 11:45 / 13:15 / 13:30 / 15:30 / 15:45 ...

손님은 자연스럽게 "11:30이 깔끔하네" 골라버림.

**그 결과:**

```
10:00  ████████████████████  손님 A
11:00  · · · ✗ 30분 데드 ✗   ← 너무 짧아서 어떤 서비스도 못 받음
11:30  ████████████████      새 손님 (숏컷 45분)
12:15  · · ✗ 15분 데드 ✗     ← 또 죽음
12:30  ████████████          손님 B
...
```

**고아 갭(orphan gap)이 두 개 생김.** 이런 갭은 거의 못 채움. 하루에 이런 일이 3~4번 일어나면 **숏컷 1개 분량의 매출이 매일 사라짐.**

## 2. 우리의 해결책 — 한 줄 공식

손님한테 가능한 시간 다 보여주지 말고, **하루를 깔끔하게 채우는 시간만** 보여주자.

```
점수 = (앞 빈칸 분) × 1.0  +  (뒤 빈칸 분) × 0.6
        ↑                       ↑
   이 슬롯이 들어가면        이 슬롯이 끝나는 시점에
   직전 예약과의 빈칸        다음 예약까지 빈칸
```

**점수가 낮을수록 좋음.** 점수가 가장 낮은 **상위 3개만** 손님한테 노출. 나머지는 숨김.

---

## 3. 왜 가중치가 다른가 (1.0 vs 0.6)

| 빈칸 위치 | 운명 | 가중치 |
|---|---|---|
| **앞 빈칸** | 두 예약 사이에 끼여서 영원히 못 채움 = 죽은 자리 | **1.0** (비싸게 침) |
| **뒤 빈칸** | 그 뒤로 워크인 / 다음 손님이 채울 수 있음 = 살릴 가능성 있음 | **0.6** (반값으로 침) |

쉽게 말하면 **"앞에 비는 건 진짜 죽고, 뒤에 비는 건 살 수 있다"**는 가정.

→ 그러니까 앞 빈칸을 더 싫어하도록 점수를 매김.

(0.6이라는 숫자는 시작값. 친구 매장 데이터 쌓이면 조정 가능. 워크인 많은 매장이면 0.4, 문자 예약만 받는 매장이면 0.8.)

## 4. 같은 예시로 점수 매겨보면

손님이 **45분짜리 숏컷** 원함. 위 캘린더에서 가능한 시작 시간 다 점수 매김:

| 시작 시간 | 앞 빈칸 | 뒤 빈칸 | 점수 | 손님한테 보임? |
|---|---|---|---|---|
| **13:15** | 0분 (12:30 예약이 13:15에 끝남) | 0분 (45분 더하면 14:00 정확히) | **0.0** | ✅ 1등 "딱 맞음" |
| **13:30** | 15분 | 0분 | **15.0** | ✅ 2등 |
| **11:00** | 0분 (10:00 예약이 11:00에 끝남) | 45분 | **27.0** | ✅ 3등 |
| 11:15 | 15분 | 30분 | 33.0 | ❌ 숨김 |
| 11:30 | 30분 | 15분 | 39.0 | ❌ 숨김 |
| 11:45 | 45분 | 0분 | 45.0 | ❌ 숨김 |

손님 화면에는 **13:15, 13:30, 11:00 세 개만** 뜸. 나머지 13개는 안 보임.

## 6. 사장(친구)이 보는 화면

지금 DropShop에서 쓰는 Approval Queue 카드 그대로 재사용:

```
┌─────────────────────────────────────────┐
│ 📅 새 예약 요청 — Sarah Park            │
│                                         │
│ 원하는 서비스: 숏컷 (45분)              │
│ AI 추천: 오후 1:15, 스타일리스트 A      │
│ 이유: 앞뒤로 빈칸 0분 — 완벽하게 채움    │
│ 손님 hold 만료까지: 2:34                 │
│                                         │
│ [ ✓ 승인 ]   [ ✗ 거절 ]                 │
└─────────────────────────────────────────┘
```

승인 누르면 → 진짜 예약으로 확정 + 같은 손님이 hold하고 있던 다른 두 슬롯은 자동으로 풀림 (다른 손님이 잡을 수 있게).

거절 누르면 → 다시 추천.

## 7. "이게 AI인가, 그냥 정렬 아닌가?"

점수 매기는 부분은 사실 단순 산수 (정렬). **AI는 그 주변 전부**:

| 단계 | AI가 하는 일 |
|---|---|
| 1. 서비스 매칭 | "숏컷"은 A·C 가능, "컬러"는 B만 가능 — 손님 자유 텍스트에서 서비스 분류 (LLM) |
| 2. 손님 이력 가중 | Sarah는 작년에 A한테 4번 옴 → B 슬롯이 0점이고 A 슬롯이 12점이어도 A를 위로 (RAG) |
| 3. 자연어 파싱 | "오후쯤, 4시 애들 축구 전에" → 12:00–15:30 윈도우로 자동 변환 (LLM) |
| 4. 컨텍스트 | 눈 오는 날, 공휴일이면 경고 / 다른 날 추천 (도구 호출) |

점수는 **결정적 핵심**. AI는 **"애초에 무엇이 후보인가"를 정하는 모든 것**.

## 8. 친구한테 60초로 설명할 때

1. "**Before**" 그림 보여줌 — 빨간색으로 고아 갭 표시.
2. "다른 앱들은 손님한테 가능한 시간 16개 다 보여줘. 손님은 깔끔한 숫자 골라. 30분 빈칸 4개 × 매일 = 숏컷 1개 매출 매일 증발."
3. "**After**" 그림 보여줌 — 카드 3개.
4. "우리 AI는 16개 중 13개를 숨겨. 하루 채우는 슬롯만 보여줘. 그게 차이야."

끝.

## 9. 수치로 환산하면 (예상)

가정:
- 친구 매장 영업시간: 10am–8pm (10시간)
- 평균 서비스 시간: 60분
- 현재 orphan gap: 일 평균 3~4개, 평균 30분
- → 일 평균 약 90~120분 = **숏컷 2~3개 분량 매출**
- 숏컷 1개 평균 가격 $40 가정 → **하루 $80~$120**, **월 약 $2,400~$3,600**

이게 우리 슬롯 추천이 친구한테 만들어주는 **실제 숫자**.
